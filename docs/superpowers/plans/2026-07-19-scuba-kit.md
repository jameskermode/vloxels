# Scuba Kit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a wearable scuba kit — a flippers pickup that grants faster, floatier underwater movement for the rest of the run.

**Architecture:** Reuse the coin pickup machinery (non-solid marker → sensor → drained overlap → cosmetic mesh). A new `wear` block flag marks a wearable; picking one up flips a persistent `wearing` flag on the player that changes only underwater physics and shows flippers under the player + a HUD chip.

**Tech Stack:** Vanilla ES modules, Three.js (InstancedMesh/Group cosmetics), Rapier (`@dimforge/rapier3d-compat`) kinematic/dynamic bodies. Node-runnable `.mjs` tests with an inline `ok()` assert, run from the repo root, importing `../src/...`. Three and Rapier both run in Node.

## Global Constraints

- Plain JS, readable for a 9-year-old; every new config value gets a comment.
- **Additive only** — do not remove or renumber any existing block id, flag, or behaviour; no level migration.
- New block: `scuba`, **id 16**, flag `wear: 'scuba'`, colour `0x11333a`, **non-solid** (no `solid` flag).
- Underwater-with-scuba tuning (exact values): horizontal `scubaSpeedMult: 1.3` (× land), idle deep-water `scubaSink: 0` (hover), swim-up `scubaSwimSpeed: 8.5`. Normal water values (`waterSpeedMult: 0.5`, `waterSink: -1.5`, `swimSpeed: 6.5`) stay as they are and apply when NOT wearing.
- Scuba affects behaviour **only while in water**; dry-land movement must be byte-for-byte unchanged whether or not the kit is worn.
- `wearing` **survives `respawn()`** (persists across deaths); it resets only by building a fresh player (entering PLAY / new run).
- Tests use real Rapier/Three in Node; run each with `node test/<name>.test.mjs` from the repo root; a suite prints `N passed, M failed` and exits non-zero on failure.

```js
// A pair of swim flippers (two flattened, slightly splayed fins). Used both for
// the world pickup and, reused, for the fins drawn under a wearing player.
export function makeFlippersMesh(color) {
  const group = new THREE.Group();
  const finGeo = new THREE.BoxGeometry(0.18, 0.06, 0.42); // width, thickness, length
  for (const side of [-1, 1]) {
    const fin = new THREE.Mesh(finGeo, new THREE.MeshLambertMaterial({ color }));
    fin.position.set(side * 0.12, 0, 0.05);
    fin.rotation.y = side * 0.2; // splay outward into a shallow V
    group.add(fin);
  }
  return group;
}
```

```js
function swimDist(wearing) {
```
